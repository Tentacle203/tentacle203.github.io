排序演算法

Jason Liao
2020-09-23
Programming

利用每周難得的自主學習課粗淺學習一下排序演算法，簡單的可以練習基本的語法和時間複雜度分析；難的可以練習邏輯與抽象思考。

附註：以 arr 作為儲存資料個數為 $n$ 的陣列，以下的實作都是把資料由小排到大。

泡沫排序

將資料分成未排序和已排序兩個部分，每次都從未排序的頭兩個資料開始倆倆比較，較大者往右挪，每次疊代比較直到已排序的部分停止。泡沫排序總共需要 $n−1$ 次疊代，每次疊代需要 $n−1−i$ 次比較（$i$ 為第幾次疊代），總共要比較 $$\sum_{i=0}^{n-1} (n - 1 - i) = \sum_{i=0}^{n-1}(n-1) - \sum_{i=0}^{n-1}i = n(n-1) - \frac{n(n - 1)}{2} = \frac{n(n-1)}{2}$$ 次。因此，時間複雜度為 $O(n^2)$。另外，疊代後若發現沒有交換的元素（資料是已排序的）那就可以提早結束排序，可以達到最佳時間複雜度 $O(n)$。

插入排序

如同整理散亂的手牌，重複地把右邊未排序的值，持續向左挪，直到適合的位置。插入排序在最好的情況下，完全不用交換，時間複雜度 $O(n)$；最壞的情況下，第 $i$ 次疊代要交換 $i$ 次，做完總共要交換 $\sum_{i=1}^{n-1}i=n(n-1)/2$ 次，時間複雜度 $O(n^2)$。

選擇排序

將資料分成已排序和未排序兩個部分，每次找未排序資料中的最小值與未排序資料前端做交換。選擇排序總共需要 $n−1$ 次疊代，每次疊代需要 $n−1−i$ 次比較（$i$ 為第幾次疊代），總共要比較 $$\sum_{i=0}^{n-2}(n-1-i) = \frac{n(n-1)}{2}$$ 次，因此無論資料是否排序完成，選擇排序仍需執行 $n(n-1)/2$ 次比較，時間複雜度為 $O(n^2)$。對於接近排序完成的資料，選擇排序並無法有自適應的方式加快排序疊代（即使資料已排序還是會進行比較）。

希爾排序

希爾排序是插入排序的改良，又叫做縮小增量排序，先選定一個步長（Gap）來分組資料，各組資料自己進行插入排序，然後再選定下一個步長，重複同樣的動作。步長並沒有一定的選法，Donald Shell 最初建議步長選擇為 $\lfloor{n/2}\rfloor$ 並且對步長取半直到步長達到 1，除此之外，也有其他更快的步長取法（序列）。在資料接近已排序好的狀況時，希爾排序的時間複雜度為 $O(n)$；而最壞的情況則會依步長取法（序列）而不同，已知最好的是 $O(n\log^2n)$，最壞的是 $O(n^2)$。

雞尾酒排序法

雞尾酒排序法又稱又叫雙向氣泡排序法、搖晃排序、波浪排序法、搖曳排序法、飛梭排序法、歡樂時光排序，是泡沫排序的改良，與泡沫排序的不同處在於排序時是以雙向在資料中進行排序。與泡沫排序相同，最好的情況下時間複雜度為 $O(n)$；而最壞的情況下，時間複雜度為 $O(n^2)$。

快速排序

用分治法（Divide and conquer），先選一個基準點（Pivot），並讓基準點左邊的資料都比基準點小；基準點右邊的資料都比基準點大，之後讓基準點左邊與右邊的資料分別都再做同樣的事直到排列完畢。基準點沒有一定的選法，但若要讓快速排序達到最佳效率，每次選的基準點要讓資料剛好分成兩份，也就是資料的中位數，在這種情況下快速排序的時間複雜度為 $O(n\log n)$；然而最壞的情況，就是選基準點的時候都選到最大值或最小值，如此無法有效地將資料分成兩個部分，最差時間複雜度為 $O(n^2)$。

合併排序

同樣用分治法，先把資料拆分成左右兩半，兩部分再分別進行合併排序，最後把排序好的資料進行合併。執行期間會把資料持續拆分到資料長度為 1，因為長度 1 的資料本身就是排序好的了。合併排序的時間複雜度是 $O(n\log n)$。

堆積排序

將資料分成未排序和已排序兩個部分，未排序的部分視為一個二元堆積，並透過 Heapify 函式將其轉換成 Max Heap，每次取 root (最大值) 放到資料後端，並把 Heap 的最後一個 node 移到 root 的位置，如此一來 Heap 的大小就 -1 了，然後重複使 Heap 變成 Max Heap，取 root (最大值) 放到資料後端，把 Heap 的最後一個 Node 移到 root 的位置…。堆積排序的時間複雜度是 $O(n\log n)$。

基數排序

基數排序是特殊的整數排序法，方法是將所有待比較的資料（正整數）照位數依次進行排序。依照位數排序的先後順序，可分為 Least significant digit (LSD) 和 Most significant digit (MSD) 兩種，代表從最低到高位和從最高位到低位。基數排序的時間複雜度是 $O(d(n+b))$，其中 $d$ 表示資料最多有幾個位數；$b$ 表示選用的基數。下面的實作是以十為基數，以 LSD 方法的基數排序。
